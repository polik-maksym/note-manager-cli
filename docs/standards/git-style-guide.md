# Git Style Guide

> Внутрішній стандарт роботи з Git у проєкті **note-manager-cli**.

## Основні правила

1.  Одна feature --- одна Git-гілка.
2.  Працюємо лише у feature-гілках.
3.  `main` завжди має залишатися стабільною.

------------------------------------------------------------------------

# Workflow

1.  Створити feature-гілку.
2.  Реалізувати функцію.
3.  Провести Code Review.
4.  Перевірити `git status`.
5.  Розбити зміни на логічні коміти.
6.  `git push`.
7.  Pull Request.
8.  Merge.
9.  Видалити feature-гілку.

------------------------------------------------------------------------

# Правила комітів

Коміт повинен описувати **одну логічну зміну**.

Приклади:

-   `Add search by tag feature`
-   `Fix input validation`
-   `Reorganize project documentation`
-   `Remove temporary test file`

Не використовувати:

-   `Update`
-   `Fix`
-   `Changes`
-   `Test`

------------------------------------------------------------------------

# Перед кожним комітом

-   Переглянути `git status`
-   Перевірити staged-файли
-   Переконатися, що в коміт входять лише потрібні зміни

------------------------------------------------------------------------

# Після Merge

``` bash
git switch main
git pull
git branch -d feature/<branch-name>
```

Після завершення роботи робоче дерево повинно бути чистим:

``` text
nothing to commit, working tree clean
```
