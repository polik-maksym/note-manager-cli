# Activity Diagram Standard

> Внутрішній стандарт проєкту **note-manager-cli**.

## Мета

Усі Activity Diagram повинні бути виконані в єдиному стилі, щоб їх було
легко читати та підтримувати.

------------------------------------------------------------------------

# Використовувані елементи

  Елемент    Призначення                       Приклад
  ---------- --------------------------------- ----------------
  Початок    Початок алгоритму                 Початок
  Кінець     Завершення алгоритму              Кінець
  Action     Будь-яка дія або виклик функції   `load_notes()`
  Decision   Умовне розгалуження (`if`)        `Знайдено?`
  Flow       Послідовність виконання           Стрілка

------------------------------------------------------------------------

# Правила

1.  Діаграма описує **логіку**, а не реалізацію Python.
2.  Одна діаграма --- одна функція.
3.  Назви дій формулюються дієсловом:
    -   Завантажити нотатки
    -   Створити список
    -   Додати нотатку
4.  Умови записуються питанням:
    -   Список порожній?
    -   Знайдено?
5.  Гілки умов позначаються **Так / Ні**.
6.  Цикли показуються поверненням потоку до початку перевірки.
7.  Параметри функції можна зазначити на початку діаграми як «Отримати
    notes та target_tag».
8.  Не показувати синтаксис Python (`for`, `append`, `return []`), лише
    логіку.
